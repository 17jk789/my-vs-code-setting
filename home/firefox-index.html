Sicherheit & Best Practices
Kompakte Empfehlungen, um JK-Arch sicher zu betreiben:
  • Regelmäßig updaten: sudo pacman -Syu
  • Pacman Keyring: sudo pacman -S archlinux-keyring falls Signaturfehler auftreten
  • Firewall: nutze nftables oder ufw und aktiviere nur nötige Ports
  • SSH-Härtung: Passwort-Login deaktivieren, SSH-Keys verwenden
  • AUR: prüfe PKGBUILD manuell; nutze vertrauenswürdige Helper (z.B. yay) mit Vorsicht
  • Backups: regelmäßige Snapshots mit timeshift oder inkrementelle Backups mit borg
  • Kernel & LTS: behalte stabile Kernel im Auge, installiere linux-lts wenn nötig

Was nicht zuverlässig funktioniert
Einige browserseitige Features sind bei `file://` eingeschränkt, z.B.:
  • AJAX/fetch zu lokalen Dateien (Sicherheitsbeschränkungen)
  • Service Worker funktionieren nicht unter file://
  • Cross-origin Einschränkungen für lokale Pfade
Wenn etwas blockiert ist
Starte bei Bedarf einen lokalen Server (nur optional):
# im Verzeichnis dieser Datei
python3 -m http.server 8000
# dann: http://localhost:8000/firefox-index.html

Arch Linux — Paketmanagement (Pacman)
Wichtigste Befehle zur Systempflege:
  • System vollständig aktualisieren: 
    sudo pacman -Syu
  • Neues Paket installieren: 
    sudo pacman -S paketname
  • Paket samt ungenutzter Abhängigkeiten entfernen: 
    sudo pacman -Rs paketname
  • Offizielle Repositorys nach Begriff durchsuchen: 
    pacman -Ss suchbegriff
  • Nicht mehr benötigte Pakete (Orphans) aufspüren und löschen: 
    sudo pacman -Rns $(pacman -Qtdq)
  • Paket-Cache komplett leeren (Speicherplatz freigeben): 
    sudo pacman -Scc
Arch Linux — AUR Helper (yay / paru)
Befehle für das Arch User Repository:
  • System inklusive AUR-Pakete aktualisieren: 
    yay -Syu
  • Paket aus dem AUR installieren: 
    yay -S aur-paketname
  • Das AUR nach einem Begriff durchsuchen: 
    yay -Ss suchbegriff
Arch Linux — Systemd (Dienste & Logs)
Hintergrunddienste steuern und Fehler finden:
  • Dienst sofort starten: 
    sudo systemctl start dienstname
  • Dienst sofort stoppen: 
    sudo systemctl stop dienstname
  • Automatischen Start bei Systemboot aktivieren: 
    sudo systemctl enable dienstname
  • Automatischen Start bei Systemboot deaktivieren: 
    sudo systemctl disable dienstname
  • Aktuellen Status und Fehler eines Dienstes prüfen: 
    systemctl status dienstname
  • System-Logs des aktuellen Boots auslesen (Fehlersuche): 
    journalctl -xb
Arch Linux — System-Wartung & Spiegelserver
Optimierung und Fehleranalyse:
  • Schnellste HTTPS-Spiegelserver via Reflector holen: 
    sudo reflector --protocol https --latest 5 --sort rate --save /etc/pacman.d/mirrorlist
  • Bremsen beim Systemstart identifizieren: 
    systemd-analyze blame
  • Festplattenbelegung übersichtlich anzeigen: 
    df -h
Arch Linux — Netzwerk & WLAN (NetworkManager)
Verbindungen verwalten über das Terminal:
  • Verfügbare WLAN-Netzwerke auflisten: 
    nmcli dev wifi list
  • Mit einem WLAN-Netzwerk verbinden: 
    nmcli dev wifi connect "SSID_NAME" password "DEIN_PASSWORT"
  • Netzwerkschnittstellen und IP-Adressen anzeigen: 
    ip a
  • Aktive Netzwerkverbindungen anzeigen: 
    nmcli connection show
Arch Linux — Hardware & Kernel-Module
System-Informationen und Treiber prüfen:
  • Aktuell geladene Kernel-Version anzeigen: 
    uname -r
  • PCI-Geräte (z.B. Grafikkarte, WLAN-Chipsatz) auflisten: 
    lspci -v
  • Angeschlossene USB-Geräte anzeigen: 
    lsusb
  • Geladene Kernel-Module (Treiber) durchsuchen: 
    lsmod | grep -i modulname
Arch Linux — Audio-Management (PipeWire)
Status prüfen und Soundprobleme beheben:
  • Status aller PipeWire-Audio-Dienste prüfen: 
    systemctl --user status pipewire pipewire-pulse wireplumber
  • Audio-Server neu starten bei Sound-Problemen: 
    systemctl --user restart pipewire pipewire-pulse wireplumber
  • Lautstärke im Terminal regeln (Mixer öffnen): 
    alsamixer
Arch Linux — Notfall-System & Chroot
Kaputtes System über einen Arch-Live-USB-Stick reparieren:
  • Root-Partition einhängen (z. B. für Reparatur): 
    mount /dev/sdXy /mnt
  • In das installierte System wechseln (Chrooten): 
    arch-chroot /mnt
  • Bootloader (GRUB) innerhalb von Chroot neu generieren: 
    grub-mkconfig -o /boot/grub/grub.cfg
